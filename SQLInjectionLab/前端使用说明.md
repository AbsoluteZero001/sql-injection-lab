# SQL 注入实验室 - 前端页面使用说明

## 📋 项目概述

这是一个用于 SQL 注入攻击模拟测试的前端界面，提供数据库登录和测试功能。

## 🚀 快速启动

### 方式一：使用启动脚本（推荐）

1. 双击运行项目根目录下的 `start.bat` 文件
2. 系统将自动：
   - 启动 Spring Boot 应用
   - 尝试自动打开默认浏览器访问 http://localhost:8080/

### 方式二：手动启动

1. 打开命令行工具，进入项目目录
2. 执行命令：`mvnw spring-boot:run`
3. 等待应用启动后，在浏览器中访问：http://localhost:8080/

## 🌐 页面功能

### 1. 登录页面 (http://localhost:8080/)

- **功能**：输入数据库用户名和密码
- **特性**：
  - 美观的渐变紫色背景
  - 响应式设计
  - 实时输入验证
  - 安全警告提示

### 2. 登录成功页面 (http://localhost:8080/success)

- **功能**：显示登录成功状态
- **特性**：
  - 成功动画效果
  - 下一步操作指引
  - 返回登录页链接

## 🔧 技术栈

- **后端框架**：Spring Boot 3.5.13
- **模板引擎**：Thymeleaf
- **前端**：HTML5 + CSS3
- **安全框架**：Spring Security

## 📁 新增文件说明

```
src/main/java/com/springboot/sqlinjectionlab/controller/LoginController.java    # 登录控制器

start.bat                                                                       # Windows 启动脚本
```

## ⚠️ 注意事项

1. **自动打开浏览器失败**：如果浏览器没有自动打开，请手动访问 http://localhost:8080/
2. **端口占用**：确保 8080 端口未被占用
3. **数据库连接**：需要 MySQL 数据库运行在 localhost:3306
4. **编码问题**：启动脚本已设置 UTF-8 编码，避免中文乱码

## 🛠️ 故障排查

### 应用无法启动
1. 检查 Java 版本（需要 Java 17+）
2. 检查 8080 端口是否被占用
3. 查看控制台错误日志

### 页面无法访问
1. 确认应用已完全启动（看到 "Started SqlInjectionLabApplication" 日志）
2. 检查防火墙设置
3. 尝试清除浏览器缓存

### 中文乱码
1. 确保使用 UTF-8 编码运行
2. Windows 系统执行 `chcp 65001` 切换代码页
